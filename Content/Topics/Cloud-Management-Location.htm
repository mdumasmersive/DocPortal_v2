Pod Location Configuration Settings

Location features in Solstice Cloud use GPS estimates that are learned from Location Services to automatically group your Pods into categories for Country, City, and Building. These categories will behave just like other categories except they’ll be populated automatically and don’t require editing. This allows a user to walk into a room, open the Solstice app, and see the Pod they are physically closest to at the top of the display list. If the automatic categories aren't intuitive enough for your users, you can assign a “friendly name” for each building which end users will see when they open the Solstice app.

Additionally, for administrative purposes, Solstice Cloud admins can easily segment groups of Pods based on their location while managing template configurations, viewing usage analytics, monitoring uptime, and much more. Using an interactive map admins can see how Pods are clustered together based on their location, and they can fine tune the location of a pod down to a few feet or even identify which floor of the building the pod is on.

The use of Location Services requires the following:

  • Pods must be imported into Solstice Cloud
  • Pods need location services enabled (Manage > Templates > Features > Room Services)
  • Pods must be running Solstice 5.1 or greater

How To

View Pod Locations
  1. In the left sidebar menu of the Solstice Cloud portal, click Manage > Location.
  2. On the bottom left-hand corner of the map, select Cluster by Pods or Cluster by Buildings.
  3. Click on one of the clusters of pods or building icons to zoom in on the map. Continue clicking on clusters until the map shows individual Pods or until you the level of detail you want to see appears.

    The Cluster by Buildings display may not represent all Pods as some Pods may not yet have a building assignment. However, all the Pods that have been imported into Solstice cloud will be represented on the map with an estimated initial location. The Cluster by Buildings display will begin to display these individual Pods when it is with two stops of maximum zoom.

  4. Click on an individual Pod to display the Pod Configuration page.

On the Pod configuration page you can perform the following actions:

  • Manually set the location for the Pod
  • Fine-tune the location of a pod
  • Add a New Building
  • Apply template configurations to the Pod.
 Manually Assign a Location for a Newly Imported Pod

To manually assign the location of a newly imported pod:

  1. In the left sidebar menu of the Solstice Cloud portal, click Manage > Pods or click Manage > Location.
  2. In the table, locate the pod whose location you want to assign, and click the Pod name.

    If you manage a large number of Pods, you can reduce the number of Pods displayed in the table by entering all or part of the Pod name in the search box or by setting a filter.

  3. Use the Set location wizard to set assign the Pod location:
    1. From the Select building drop-down menu, select the building, and then click Next.

    2. If the desired building is not in the list add a new building. (For details on how to do this see How to Add a Building.)

      The new building now appears in the Select Building field.

    3. Click Next.

The location is set for the Pod, and the map zooms in to the Pod's location.

 Manually Change a Pod Location
  1. In the left sidebar menu of the Solstice Cloud portal, click Manage > Pods or click Manage > Location.
  2. In the table, locate the pod whose location you want to assign, and click the Pod name.

    If you manage a large number of Pods, you can reduce the number of Pods displayed in the table by entering all or part of the Pod name in the search box or by setting a filter.

  3. Click in the Building field. (Do not click the drop-down menu.)

    The field will change to a search box, and you can start typing the building name in the field.

  4. From the filtered building list that appears, select the desired building.
  5. If the desired building is not in the list add a new building. (For details on how to do this see How to Add a Building.)
  6. If there a multiple floors in a building, use the drop-down to select the floor that the Pod is on. If the correct floor does not show up on the list, click Add New Floor and enter the floor number. Then click Save.

The location is set for the Pod, and, if the change is location is significant, the map zooms in to the Pod's location.

Fine-tune a Pod Location

When the map is zoomed in on the Pod's location, you can “fine tune” the location if it is not accurate.

To fine tune a pod location:

  1. Click Fine-tune Location.

    The cursor changes to a hand icon.

  2. Click in the map area and drag the map around as necessary until the Pod is in the exact location.
  3. Click Save.

The Pod now displays at the correct position on the map.

After you have fine-tuned a Pod location, Location Services will no longer attempt to refine the location of the Pod.

 Add a Building

If the building where the Pod is located does not exist in a Building drop-down list, you can add a new building:

  1. At the bottom of the Building drop down list, click Add New Buidling.
  2. In the Building Address field, start typing the building address, and from the filtered address list that appears, select the appropriate address.

  3. If desired, in the Building Name field, enter a common or "friendly name" for the building.
  4. Click Save

The new building appears in the Select Building feld.

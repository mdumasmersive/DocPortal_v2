Step 2: Create Base Configuration Templates

Solstice Cloud allows administrators to create unlimited configuration templates to apply across their Solstice deployment. The following are the recommended base configuration templates and settings and are intended to be used as a guide to help you get Solstice quickly deployed. The settings you ultimately need depend on your organization's needs and IT security policies. For network, please consult with IT or network team. For our full list of templates and configurations, see Solstice Cloud: Configuration Templates.

Mersive recommends updating each template category's Default template. However, you can also create new templates if you wish to have different configurations based on office location, department, etc.

Welcome Screen Template

To make it easy for users to discover and connect to the right Solstice display, Mersive recommends renaming the Pod to correspond to the meeting room or space it will be installed in. You can also change the appearance of the Solstice display's welcome screen to match your organization's branding by updating the display's background images, adding customized connection instructions, changing the text color, and more.

  1. In the left sidebar navigation panel of Solstice Cloud, click Manage > Templates > Welcome Screen.
  2. Click Create New Welcome Screen Template.
  3. Enter the Template Name.
  4. To change background images, click on the upload icon for that background image. Then browse to the image file and select.
  5. To disable background images from the rotating display, deselect the checkbox in the lower right-hand corner of the background image preview. 
  6. To customize the connection instructions that display on the welcome screen, go to the Instructions drop-down and select how you would like the connection instructions to display on the Welcome Screen for end-users.
    • Default: This setting uses Solstice's default connection instructions.  
    • Show custom instructions: Allows you to create custom instructions for meeting attendees to see on the Welcome Screen.  
  7. If you selected the Show custom instructions option, a rich text field appears below, allowing you to enter and format custom connection instructions. Use responsive variables such as [RoomName] and [ScreenKey] that will be auto-populated with the Pod-specific data.
  8. To hide the Airplay instructions if you are not enabling sharing via AirPlay, deselect the Show Airplay instruction line checkbox.
  9. To hide the Miracast instructions if you are not enabling sharing via Miracast, deselect the Show Miracast instruction line checkbox.
  10. Click Save.
Calendar Template

Integrating a room calendar allows you to display the schedule and calendar information for the room when there is no other content being shared. Participants can see if the space is currently scheduled or available, as well as the next three upcoming meetings in the space.

  1. In the left sidebar navigation panel of Solstice Cloud, click Manage > Templates > Calendar.
  2. Click Create New Calendar Template.
  3. Enter the Template Name.
  4. Select the Enable Calendar Feature checkbox.
  5. From the Calendar Type drop-down, select the type of calendar you are integrating [for the room]: Microsoft Exchange, Office 365, or 3rd Party Only. Only select 3rd-party only if you are using Solstice’s OpenControl API to integrate a third-party calendar.
  6. If you are using Microsoft Exchange server, enter the Microsoft Exchange server URL in the Server URL field.
  7. In the Authentication type drop-down, select the type of authentication your server is using: Basic or NTLM.
  8. If you are a using either an Impersonation or a Delegation Mailbox, select the corresponding radio button.
    • If you selected No, skip to step 6.
    • If you selected either Impersonation or Delegation, you will need to (required) enter in the user name and password for the impersonation or delegation account.

      The login credentials for email accounts will need to be entered individually for each Pod. These steps are covered in Step 5: Applying Individual Pod Settings.

  9. If you wish to hide meeting titles or meeting organizers from being visible on the room display, deselect Show meeting titles and/or Show meeting organizers.
  10. From the Update Interval drop-down, select the frequency at which the Pod will update the calendar meeting information visible on the display.
  11. Click Save.
Features Template

These settings determine which app-free sharing options users will be able to use to share content to the Solstice display. This template is also used to enable use of screen key for added security for Solstice collaboration sessions.

  1. In the left sidebar navigation panel of Solstice Cloud, click Manage > Templates > Features.
  2. Click Create New Features Template.
  3. Enter the Template Name.
  4. Under the Solstice Feature Options section, select Enable screen key. This is an important security setting that requires end-users to enter the 4-digit screen key visible on the display to connect and share to Solstice.
  5. In the same section, enable or disable the various sharing app-free options based on your organization's needs and policies.
    • Miracast WiFi Direct: Allows users to mirror their Windows device screen; streams P2P from the Windows device to the Pod.
    • Miracast Infrastructure: Allows users to mirror their Windows device screen; streams over the existing network.
    • Android mirroring: Allows users to mirror their Android device screen.
    • iOS mirroring (AirPlay): Allows users to mirror their iOS device screen.
      • AirPlay discovery proxy: Enable this option if your network does not allow use of Apple's Bonjour.
      • Bluetooth discovery for AirPlay: Provides another alternative for discovery for environments that do not allow UDP broadcast traffic or Apple's Bonjour protocol by allowing AirPlay users to discover the Pod using Bluetooth. Users will still have to connect to the same network as the Pod in order to stream content via AirPlay.
    • Clientless sharing via browser: Allows users to connect and share content via a web browser without the Solstice App.
  6. Click Save.
 Room Intelligence Template

These settings allow your Solstice configuration to sense and adapt to the room in which it is installed, integrating any conferencing solution installed on the host's laptop (Zoom, Teams, GoToMeeting, etc.) and automatically detecting useful information about a conference room.

  1. In the left sidebar navigation panel of Solstice Cloud, click Manage > Templates > Room Intelligence
  2. Click Create New Room Intelligence Template.
  3. Enter the Template Name.
  4. Select Enable Solstice Conference if you wish to enable the Solstice Conference capability.
  5. Click Save.
 Ethernet Template

Solstice is designed to leverage existing Ethernet networks to support collaboration in meeting rooms and learning spaces. Mersive recommends connecting your Pod to Ethernet for best performance.

These steps outline how to configure basic Ethernet network settings to apply across your deployment. For more advanced Ethernet settings, see Solstice Cloud: Ethernet Template.

  1. In the left sidebar navigation panel of Solstice Cloud, click Manage > Templates > Ethernet.
  2. Click Create New Security Template.
  3. Enter the Template Name.
  4. To activate the options, select the Enable Ethernet checkbox.
  5. Change the Network Name to the one that users will see in the list of available networks on their device so that it is easily recognizable.
  6. If you wish to allow admin access to make configuration changes on this network, select the Allow administrative configuration access checkbox.
  7. Select either DHCP for the Pod to be dynamically assigned an IP address, or select Static IP to enter your network configuration manually.
  8. If you selected Static IP, enter the Gateway, Network Prefix Length and DNS 1 fields. Because it is unique to each Pod, you will enter the IP Address after this template is applied to a Pod.

    9. Note that the IP Address is unique to each Pod, and can be added only after this template is applied to a Pod. This is covered in Step 5: Applying Individual Pod Settings.

  9. Click Save.
 WiFi Template

Solstice is designed to securely leverage a wireless network connection that can be applied as Solstice's only network connection, or in dual-network mode alongside a wired Ethernet connection.

These steps outline how to configure basic WiFi network settings to apply across your deployment. There are two options outlined below: Option 1. Attach the Pod to a Wireless Network, Option 2. Disable WAP Mode. If you are not attached the Pod to a wireless network, Mersive strongly recommends following the steps for Option 2 to disable WAP mode. For more advanced WiFi settings, see Solstice Cloud: WiFi Template.

Option 1. Attach the Pod to a Wireless Network

  1. In the left sidebar navigation panel of Solstice Cloud, click Manage > Templates > Ethernet.
  2. Click Create New Security Template.
  3. Enter the Template Name.
  4. Select the Enable WiFi checkbox.
  5. If you wish to allow admin access to make configuration changes on this network, select the Allow admin configuration access checkbox.
  6. Select Existing Network. This option connects the Pod to an existing network wirelessly.
  7. Enter the network name in the SSID field.
  8. In the Security Type drop-down, choose the appropriate option: Open, WEP, WPA/WPA2, or 802.1x EAP.

    9. If you chose 802.1x EAP, see Solstice Cloud: WiFi Template for more information on this configuration.  

  9. If prompted, enter the Password for this network.
  10. Select either DHCP for the Pod to be dynamically assigned an IP address, or select Static IP to enter your network configuration manually.
  11. If you selected Static IP, enter the Gateway, Network Prefix Length and DNS 1 fields.

    12. Note that the IP Address and DNS Hostname are unique to each Pod and need to be configured for every Pod. Once this template is applied to your Pods, you be will prompted to enter this information for the each Pod the template is applied to.

  12. Click Save.

Option 2. Disable WAP Mode

  1. In the left sidebar navigation panel of Solstice Cloud, click Manage > Templates > Ethernet.
  2. Click Create New Security Template.
  3. Enter the Template Name.
  4. Deselect the Enable WiFi checkbox.
  5. Click Save.
Security Template

Before deploying your Solstice Pods, certain security baselines should be configured to harden the security of your deployment. The following are the base security settings that Mersive recommends.

  1. In the left sidebar navigation panel of Solstice Cloud, click Manage > Templates > Security.
  2. Click Create New Security Template.
  3. Enter the Template Name.
  4. If you wish to set an admin password to protect configuration, enter in the password you will use to access the Solstice configurations. It is highly recommended that you set the same administrator password for all of your Pods.
  5. If you wish to enforce password validation rules (8-character minimum, one uppercase and one lowercase character, one number or special character), select the Enforce password validation rules option.
  6. Ensure the Enable encryption for Solstice traffic option is selected.
  7. If you wish to upload a custom TLS certificate to be used instead of the Pod's default self-signed certificate, click the Upload Certificate button, then browse to and select the certificate file.
  8. Click Save.
 Time/Locale Template

The Time/Locale Template page allows you to set various system preferences for your Solstice display, including timezone and language settings.

  1. In the left sidebar navigation panel of Solstice Cloud, click Manage > Templates > Time/Locale.
  2. Click Create New Time/Locale Template.
  3. Enter the Template Name.  
  4. To set the date and time using a time server, Enable the Set date/time automatically checkbox and enter the time server URL in the corresponding field (the default timeserver URL is pool.ntp.org).
  5. If you want the time to display in 24 hour format (4:00 pm is displayed as 16:00), select the 24 hour format checkbox.
  6. From the Timezone drop-down, select the timezone the Pod is in (for example, Eastern Time).
  7. From the Language drop-down, select the language you would like to display on the Pod.
  8. Click Save.

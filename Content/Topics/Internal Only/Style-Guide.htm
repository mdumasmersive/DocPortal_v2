Formatting

Standard Topic Sections

  • Topic Title/Overview [h1]
  • Sub-section with additional information [h2]
  • How To section [h2] (note: if single how-to, make full title "How to Do Example Task" h3]
  • FAQ section [h2]
  • Related Topics (or if in sequential step-by-step guide, add a Next Topic section) [h4]

See the Topic Title on page 1 for an example topic structure.

Notes, Tips, Warnings

The CSS stylesheet for this project (styles.css) has 3 additional paragraph <p> style classes to help distinguish important information such as notes, tips, or warnings that we do not want to get lost in the main paragraph or how-to.

Notes are used to communicate important information about a feature or workflow to the admin or end-user that we do not want them to miss.

Tips are used to call out best practices or additional tips that might make the process easier for the reader. For example: In Solstice Cloud, you can assign your Pods to categories, such as location or floor level, to make configuration easier.

Warnings are used to communicate critical information, typically to administrators, to let them know if there are any caveats or mis-steps that might negatively affect their Solstice deployment or the functionality of the product.

To apply a note, tip, or warning to the text:

  1. Ensure the text has its own <p> line. This can be visually confirmed in the XML Editor view of the topic as the line should have a 'p' node visible on the left-hand side of the topic.
  2. Click the line's 'p' node to select the entire line.
  3. In the Flare toolbar menu, go to Home > Style drop-down, then select either p.note, p.tip, or p.warning.

Drop-down Text

Drop-downs are typically used within a topic for three things:

  • When there are two or more how-to's
  • When there is one or more FAQ
  • Within quickstart or deployment topics, to conceal dense and technical information

Capitalization

Capitalize all proper names, including 'Pod', and all product names.

When referring to GUI elements, match the capitalization used in the interface.

Use title-case capitalization for headings.

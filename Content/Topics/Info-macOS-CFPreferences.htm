Configuring the Solstice App with CFPreferences on macOS

Preferences for the Solstice app on macOS can be configured using the Settings screen in the Solstice app or by configuring a Solstice plist (property list) using the macOS command line utility defaults.

Solstice App Settings for macOS

The following client options can be set using the defaults utility.

Both the Solstice app and defaults write to the plist. If a user changes a setting in the client, a value that was set using defaults may be overwritten, and vice-versa.

Option Type Description
PROFILE/AUTOSHAREDESKTOP Bool If set to TRUE, Solstice will automatically share your desktop to the Solstice display once you connect to the Pod.
PROFILE/AUTOSHAREAUDIOWITHDESKTOP Bool If set to TRUE, Solstice will automatically stream your desktop audio to the Solstice display when you share your desktop
PROFILE/CALENDAR_CONFIG/REFRESH_TOKEN Bool If set to TRUE, an Office 365 calendar has been integrated with the Solstice app. You can set the value to FALSE to terminate the calendar integration; however, the calendar integration needs to be configured in the Solstice app settings in order to authenticate with Office 365.
PROFILE/CPUWARNINGCHECKCOUNT Uint32 Reserved.
PROFILE/CPUWARNINGCHECKINTERVALMS Uint32 Reserved.
PROFILE/CPUWARNINGFAILPERCENTAGE Uint32 Reserved.
PROFILE/CPUWARNINGTOTALUSAGETHRESHOLD Uint32 Reserved.
PROFILE/HIDEUIWHENSHARING Bool If set to TRUE, the Solstice app will automatically hide when you share your desktop.
PROFILE/ISFIRSTRUN Bool When the Solstice app is first installed, this option is set to TRUE. This causes the Solstice app to start with a "get started" screen that prompts the user to enter a profile name. After the app is first run, this value will be set to FALSE.
PROFILE/ISFIRSTTIMEINMINUI Bool When the Solstice app is first installed, this option is set to TRUE. This causes the Solstice app to display a "got it" prompt the first time the client application is automatically hidden or minimized. (See PROFILE/HIDEUIWHENSHARING.) After the app is first run, this value will be set to FALSE.
PROFILE/ISFIRSTWALKTHROUGHRUN Bool When the Solstice app is first installed, this option is set to TRUE. This causes the Solstice app to display a "walk through" animation about how to connect your client to a display. After the app is first run, this value will be set to FALSE.
PROFILE/MANUALMRUDISPLAYS String ???
PROFILE/MANUALSDSADDRESS String If your organization has implemented the Solstice Discovery Service (SDS), the SDS IP address is set here in order for the Solstice app to display available displays to connect to.
PROFILE/NAME String How your name will appear in the collaboration session.
PROFILE/NOTIFICATIONLEVEL Uint32

Set which notifications you would like to receive:
0 – Do not display any notifications.
1 – Display only critical notifications.
2 – Display all notifications.

PROFILE/PINGWARNINGCHECKCOUNT Uint32 Reserved.
PROFILE/PINGWARNINGCHECKINTERVALMS Uint32 Reserved.
PROFILE/PINGWARNINGFAILPERCENTAGE Uint32 Reserved.
PROFILE/PINGWARNINGLATENCYTHRESHOLD Uint32 Reserved.
PROFILE/RELAYSHORTCUT String This setting allows you to configure the shortcut command to quickly open the Solstice Conference pop-up without having to click the icon in your system tray. The default setting is C, which corresponds to Cmd+Opt+C.
PROFILE/RELAYWARNINGDURATION Uint32 Reserved.
PROFILE/RELAYWARNINGINTERVALMINS Uint32 Reserved.
PROFILE/UUID String Reserved.

Examples of Using Defaults on macOS

Here is an example of how to read the values in the Solstice app plist:

$ defaults -currentHost read com.mersive.solstice.client

If a settings option is not currently set in the plist, you can create it by specifying the key type and key value. For example, to configure the Solstice app to automatically share the desktop with the display when it connects, you could enter this:

$ defaults -currentHost write com.mersive.solstice.client "PROFILE/AUTOSHAREDESKTOP/TYPE" Bool
$ defaults -currentHost write com.mersive.solstice.client "PROFILE/AUTOSHAREDESKTOP/VALUE" T

Here is an example of how to change a configure setting for the profile name to Jerry:

$ defaults -currentHost write com.mersive.solstice.client "PROFILE/NAME/VALUE" "Jerry"

Here is an example of how to delete a configured setting in the Solstice app plist:

$ defaults -currentHost delete com.mersive.solstice.client "PROFILE/AUTOSHAREDESKTOP/TYPE"
$ defaults -currentHost delete com.mersive.solstice.client "PROFILE/AUTOSHAREDESKTOP/VALUE"

The Solstice app reads the plist settings on start up. If you change plist values using the defaultscommand while the app is running, the app will not show the change on the Settings page. However, the app will behave in accordance with the new plist setting.
For example, if the plist setting for AUTOSHAREDESKTOP is FLASE, then the Solstice app will show that Quick Share Desktop is OFF. However, if the value is changed to TRUE using the defaults command, the Settings page does not change; however, when the app connects to the display the desktop will be automatically shared. To force the Solstice app to change what is displayed on the Settings page, you will need to exit and restart the app.

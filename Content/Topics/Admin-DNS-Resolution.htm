DNS Resolution

Summary paragraph about what DNS is and why it's important to Solstice.

In order to utilize DNS resolution for your Solstice display, there are two methods:

If your networks utilize one DNS server

If your networks utilize multiple DNS servers

-automatic DNS resolution within Solstice. If all network interfaces you want to use DNS for share the same DNS server, you can have Solstice automatically resolve and display the DNS entries for each network interface on the welcome screen. This requires editing the hostname value (system tab dashboard) to match your desired DNS hostname. If utilizing static IP addresses, IT admins will need to create the DNS entry that matches that same hostname value on their internal DNS server. Be sure to create a reverse DNS entry for the Pod to be able to find its hostname. This hostname will then be passed to your DNS server from each network interface. If you enable display DNS hostname (display tab), you can display short or fully qualified domain names. this will only work if the DNS value returned matches on all network interfaces.

Mersive highly recommends using SDS if implementing DNS resolution.

^^ change/reference the hostname. select the DNS option. if static IP, enter each network interface, add the internal DNS server IP address.

-manual entry of DNS hostname into Pod/Dashboard for each connected network interface. Must be applied individually and will differ from Pod to Pod due to unique hostname. Can differ from interface to interface if utilizing different domain names. This requires admins to completely configure all DNS entries on their network prior to entering the desired display hostname in Dashboard.

If you wish to utilize DNS resolution and have added a DNS entry in your DNS server to resolve to the Pod's IP address, you can enter in the DNS Hostname (for example, hostname.domain) that you wish to show on the display's welcome screen. For more information on DNS resolution, see DNS Resolution.

^^ enter in the hostname. in network, make sure dns entry in dns server is able to resolve to the pod's IP address.

Enable “Use DNS for my welcome screen if DNS returns successfully” so that users will enter the DNS name and not an IP Address to QuickConnect.

DNS resolution is only supported when all interfaces utilize the same DNS server. The A records have to match for each network interface in order for us to display a single DNS entry that can be used across all interfaces.
